# 흰 배경 제거 도구 및 크롭 (white-bg-remover)

이미지를 업로드하면 중앙 요소는 그대로 두고 바깥쪽 흰 배경만 투명하게 지운 PNG를 만들어 주는 도구입니다.
단일 HTML 파일(`index.html`)로 되어 있고, 외부 라이브러리·서버 없이 브라우저 안에서만 동작합니다
(이미지는 어디에도 전송되지 않습니다).

## 사용 방법

- **가장 간단하게**: `index.html`을 더블클릭해 Chrome 등 브라우저에서 열면 끝.
  일반 브라우저에서는 저장 버튼이 다운로드 폴더로 바로 저장합니다.
- **Cowork 아티팩트로**: Claude 데스크톱 앱 사이드바의 `white-bg-remover` 아티팩트로도 동일한 도구가 등록되어 있습니다.

## 주요 기능

- 여러 장 동시 업로드(파일 선택 · 드래그 · 붙여넣기), 이미지별 원본/결과 카드 미리보기
- 가장자리와 이어진 흰색만 제거(기본, 요소 내부 흰색 보호) 또는 모든 흰색 제거
- 허용 오차 슬라이더(기본 10) — 배경이 순백이 아니거나 JPG 노이즈가 있을 때 조절
- 가장자리 부드럽게 + 색 보정: 경계 픽셀이 맞닿은 안쪽 요소 색을 그대로 물려받아(color bleed)
  흰 테두리(halo) 없이 깔끔한 외곽선 생성
- 요소에 맞게 크롭(체크박스, 기본 켜짐): 배경 제거 후 남은 요소의 경계 상자에 딱 맞게 자동 크롭
- 긴 면 1000px(체크박스): 비율을 유지한 채 긴 쪽이 1000px이 되도록 리사이즈 (크롭과 조합 가능)
- 크롭 수치 표시: 카드마다 요소 경계 상자의 위치·크기·원본 대비 비율·4방향 잘리는 여백을 숫자로 표시
- 원본 미리보기 클릭 시 붉은 크롭 보조선: 원본 위에 크롭 영역 사각형 + 점선 연장선 + 바깥 음영 토글
- 결과 미리보기 클릭 시 상세보기 팝업: 결과를 크게 확인하면서 광택 제거·색 보정 토글과
  보정 강도 슬라이더를 그 자리에서 조절(실시간 반영, 카드와 완전 동기화), PNG 저장·ESC/배경 클릭 닫기
- 광택 선 제거(이미지별 토글 버튼): 요소 내부의 흰 광택 하이라이트를 감지해 주변 색으로 인페인팅
  (onion-peel 채움 + 잔광 링 색 복원 + 스무딩). 이미지마다 켜고 끌 수 있으며 채운 픽셀 수를 표시
- 색 보정(이미지별 토글 버튼): 검정·회색이 번진 픽셀을 감지해 주변에서 가장 많이 쓰인 단색으로 채움.
  양자화 팔레트에서 무채색·탁한 변형 톤을 걸러내고, 최빈색 투표(onion-peel)로 단색 스냅 채움.
  단색 위주 이미지에 적합 (그라데이션 이미지에서는 평탄화될 수 있음).
  카드별 "보정 강도" 슬라이더(0~100, 기본 10)로 감지 범위를 조절 — 낮음: 확실한 번짐만,
  높음: 생성형 이미지의 저대비 일렁임(옅은 회색·검정)까지 폭넓게 보정.
  감지 규칙 2종: (a) 채도가 급감한 픽셀(검정·회색이 '섞인' 번짐),
  (b) 색조는 같은데 밝기만 낮아진 픽셀(검은 기운이 '덮인' 일렁임 잔상 — 채도 유지형).
  추가로 팔레트 색을 소속 픽셀들의 실제 평균으로 정밀화한 뒤, 팔레트에 속한 모든 픽셀을
  그 원색으로 스냅(평탄화)해 허용 오차 안쪽에 숨은 미세한 일렁임·명암 물결까지 완전 제거.
  팔레트 선정은 "연속체 판정"(두 색 사이 중간색 픽셀 밀도)으로 번짐의 중간 톤과
  진짜 소수 단색(예: 과일 단면의 밝은 테두리)을 구분 — 면적 0.3%의 작은 단색도 보호
- 저장: 개별 PNG 저장 / 전체 저장. 임베드(사이드바) 환경에서는 최초 1회 저장 폴더를 허용하면
  이후 저장은 대화상자 없이 그 폴더에 바로 기록되며, 기록 후 파일을 다시 읽어 크기까지 검증
- 저장이 차단되는 환경을 위한 자동 폴백: 파일별 저장 대화상자 → 일반 다운로드 → 새 창 → 저장 패널
- 결과·실패 원인을 화면 상태줄에 항상 표시

## 알아두면 좋은 점

- 브라우저 보안 정책상 "다운로드" 폴더 자체는 폴더 선택에서 거부될 수 있습니다.
  그 경우 다운로드 폴더 안에 하위 폴더(예: `투명PNG`)를 만들어 선택하세요.
- 아티팩트 사이드바처럼 샌드박스된 환경에서는 blob: URL과 자동 다운로드가 차단되기 때문에
  코드 전반이 data: URL과 File System Access API를 사용하도록 작성되어 있습니다.

## 처리 알고리즘 (index.html 내 processImageData)

1. 가장자리에서 시작하는 flood fill(BFS)로 배경 흰색 영역을 찾고 알파 0 처리
2. 경계 밴드(제거 영역과 맞닿은 1픽셀 + 안쪽 1픽셀) 계산
3. 안쪽 밴드 색을 순수 내부 픽셀 평균으로 정리하고, 바깥 밴드는 그 색을 물려받음
4. 바깥 밴드의 투명도를 흰색 근접도에 따라 부드럽게 부여
5. (옵션) 알파>0 픽셀의 경계 상자로 크롭 → (옵션) 긴 면 1000px로 비율 유지 리사이즈

## 테스트

Playwright 기반 자동 테스트가 `tests/`에 있습니다.

```bash
npm install          # playwright 설치 (최초 1회, 브라우저 다운로드: npx playwright install chromium)
npm test             # 저장 시나리오 5종(성공/재선택/차단/전부차단/취소) 11개 검사
npm run test:multi   # 다중 업로드·처리·설정 변경·카드 제거
npm run test:download# 최상위/샌드박스별 다운로드 경로
npm run test:remember# 저장 폴더 기억(새로고침 후 무대화상자 저장)
npm run test:crop    # 크롭·1000px 리사이즈 조합 9개 검사
npm run test:guides  # 크롭 수치 표시·붉은 보조선 토글 10개 검사
npm run test:gloss   # 광택 선 제거 토글 10개 검사
npm run test:correct # 색 보정(번짐 채움) 토글 11개 검사
npm run test:strength# 보정 강도 슬라이더 8개 검사
npm run test:detail  # 상세보기 팝업 11개 검사
npm run test:darkwave# 같은 색조 어두운 잔상 보정 5개 검사
npm run test:flatten # 3종 잔상 + 완전 평탄화 스트레스 10개 검사
npm run test:guava   # 소수 단색(링) 보호 + 보정 공존 9개 검사
```

시스템에 설치된 Chromium을 쓰려면 `CHROMIUM_PATH=/path/to/chromium npm test` 처럼 지정할 수 있습니다.

